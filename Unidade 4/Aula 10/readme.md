## Objetivos da aula:

Compreender a evolução das redes de computadores.
Identificar os elementos básicos de uma rede.
Classificar redes segundo abrangência geográfica e modelo computacional.
Roteiro da aula:
## 1. Introdução 

Breve explanação sobre a evolução das redes (dos mainframes à ARPANET e Internet).
Apresentação dos quatro elementos básicos de uma rede (emissor, receptor, meio de transmissão, protocolo).
Orientação sobre a atividade prática.

## 2. Atividade Prática em Grupos 

Cada grupo deverá realizar as seguintes tarefas:

Parte A – Linha do Tempo 

Criar uma linha do tempo ilustrada mostrando os principais marcos da evolução das redes (ex.: mainframes, ARPANET, TCP/IP, Internet comercial, redes móveis).
Pode ser feita em cartolina, quadro ou ferramenta digital (Google Slides, Canva).
Parte B – Elementos da Rede 

Montar um esquema simples (desenho ou diagrama digital) representando os quatro elementos básicos de uma rede.
Exemplificar com dispositivos reais (ex.: notebook como emissor, roteador como meio de transmissão, servidor como receptor, protocolo HTTP).
Parte C – Classificação de Redes 

Escolher exemplos reais para cada tipo de rede:
PAN: Bluetooth entre celular e fone de ouvido.
LAN: Rede doméstica ou de laboratório.
MAN: Rede de uma universidade ou prefeitura.
WAN: Internet.
Representar em um quadro comparativo.

## 3. Apresentação dos Grupos 

As produções (linha do tempo, esquema e quadro comparativo) devem ser publicadas no repositório GitHub da disciplina, em uma pasta específica para a Aula 10.

## 4. Síntese e Fechamento

Comparação entre as classificações apresentadas.
Discussão coletiva: “Como a evolução das redes impacta nossa vida cotidiana e profissional?”
Entregáveis da Aula
Produção em grupo: Linha do tempo + esquema dos elementos + quadro comparativo das redes, publicados no GitHub da disciplina.
Participação na aula: Anotações individuais como bloco de notas.
Reflexão individual (para casa): Texto curto (1 página) sobre como a evolução das redes influenciou a sociedade. (Bom para a avaliação 2)
Atenção: usar o modelo de README.md ao subir os trabalhos no GitHub, com instruções de organização e documentação dos arquivos.
